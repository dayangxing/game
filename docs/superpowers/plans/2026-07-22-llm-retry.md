# LLM 调用失败重试 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为所有 Bailian LLM 调用增加最多 3 次失败重试，并让流式重试不会把两次响应拼接在一起。

**Architecture:** 在 Bailian 客户端集中判断可重试错误并执行首次调用加 3 次重试；流式请求通过回调向 storyDirector/普通 narration 传播 reset 事件，后端 SSE 转发，前端清空当前半成品后继续显示新尝试。规则结算和 fallback 行为保持不变。

**Tech Stack:** Node.js ESM、原生 `node:test`、Fetch、ReadableStream、SSE、Svelte 5。

## Global Constraints

- 重试次数：首次调用失败后额外重试 3 次，最多 4 次调用。
- 重试间隔：500ms、1000ms、2000ms；测试注入等待函数，不真实等待。
- 可重试错误：网络异常、HTTP 408/409/425/429 和 5xx。
- 不可重试错误：缺少 API Key、fetch 不可用、鉴权失败及其他 4xx。
- 流式中断在已发送片段后必须先 reset，再发送下一次尝试的片段。
- 重试耗尽后保持既有 fallback 与 `LLM_UNAVAILABLE` 语义。
