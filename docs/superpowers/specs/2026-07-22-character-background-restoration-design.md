# 角色背景设定恢复设计

## 目标

恢复正式角色创建阶段的角色背景可见性，并让连续剧情导演稳定获得角色的完整背景设定。身世、灵根、命格天赋仍由后端按角色 seed 确定；玩家手动分配的五维属性继续保留。

## 范围

- 新增一个无副作用的角色预览接口，使用正式创建相同的后端随机逻辑。
- 创建角色页展示预览的出身、灵根和命格天赋，同时继续展示五维属性分配。
- `storyDirector` prompt 明确接收身世、灵根和命格天赋。
- 为预览接口、prompt 上下文和前端调用增加回归测试。

不在本次范围内：把身世、灵根或命格改为玩家手动编辑；修改角色随机池；改变正式创建或回合结算规则。

## 设计

### 1. 权威角色预览

新增 `POST /api/v1/game/character-preview`，请求体使用 `{ rerollSeed, name, attributes }`。接口调用现有 `rollCharacter`，只返回角色预览，不替换 `state.game`，不修改回合、行动、日志或存档。

正式创建仍然调用同一个 `rollCharacter`。因此，只要 seed 和属性分配不变，预览中的身世、灵根、命格与正式创建结果一致。

当后端不可用时，前端 API 层使用现有 mock 角色生成逻辑提供降级预览；预览失败不会阻止玩家创建角色。

### 2. 前端状态与界面

角色创建 store 保存 `_pendingCharacterPreview`。初始化角色创建页和点击“随机分配”后刷新预览；调整五维属性只更新待提交属性，背景字段不变。

`CharacterCreation.svelte` 增加角色背景卡片，展示：

- 出身
- 灵根
- 命格天赋

背景卡片在预览尚未返回时显示加载文案；预览失败时显示“入山门后揭晓”，不暴露错误，也不阻塞开始修行。

### 3. LLM 上下文

`buildStoryDirectorMessages` 的 context 增加 `characterBackground`：

```json
{
  "origin": "渔村遗孤",
  "spiritualRoot": "土灵根",
  "traits": ["经脉坚韧", "福缘深厚"]
}
```

字段来自后端权威 `game.character`。prompt 约束要求剧情不得无依据改写这些既定背景；背景只作为叙事事实，不赋予 LLM 修改规则状态的权限。

### 4. 错误处理

- seed 或属性不合法时沿用角色创建的校验错误格式。
- 预览接口异常不改变游戏状态；前端保留当前预览或显示降级文案。
- 正式创建仍以 `/game/new` 的后端结果为准，不能提交前端自行修改的背景字段。

## 验证

- 后端测试预览结果与正式创建结果在相同 seed 下的背景字段一致。
- 后端测试预览不会改变游戏 turn、version、character 或 pending actions。
- prompt 测试确认身世、灵根、命格进入导演上下文。
- 前端 API 测试确认预览请求体和返回值。
- 前端构建通过；核心 API 与联调测试通过。
