# 文字修仙本地 Web 原型设计

## Goal

构建一个可在本地浏览器运行的大模型驱动文字修仙游戏原型。首版突出自然语言行动、随机世界演化、角色状态变化、NPC 记忆和剧情日志；没有 API key 时使用本地 mock 叙事，有 API 配置时预留 OpenAI-compatible 接入位置。

## Product Shape

首屏就是可玩界面，不做营销页。页面采用三栏布局：

- 左侧：玩家角色、境界、灵根、寿元、灵石、灵气、心境、宗门关系、NPC 好感。
- 中间：剧情日志、行动建议、自然语言输入框、模型模式切换。
- 右侧：世界演化时间线、长期伏笔、NPC 记忆、导出和重开操作。

## Core Loop

玩家输入自然语言行动后，系统执行一次回合推演：

1. 本地规则引擎识别行动意图。
2. 根据角色状态、地点、随机种子和意图更新资源、境界进度、NPC 关系和世界事件。
3. mock 叙事器生成剧情文本、NPC 反馈、后续建议。
4. 页面把回合写入日志，并保存到 `localStorage`。

## Architecture

原型使用无构建依赖的静态 Web 技术，便于快速运行和迁移：

- `src/engine.js`：纯函数游戏规则、状态初始化、回合推进、导出小说。
- `src/app.js`：浏览器状态管理、DOM 渲染、事件绑定、存档。
- `src/styles.css`：完整产品界面样式。
- `tests/engine.test.js`：Node 内置测试，覆盖核心推演行为。

## Constraints

- 必须能在没有网络、没有包安装的环境下运行。
- 原型以 mock 叙事为默认模式，界面保留 API 模式入口但不在前端暴露真实密钥流程。
- 随机演化必须保持规则自洽：境界、资源、NPC 记忆和事件日志由规则引擎控制。
- 中文 UI 文案，暗色水墨修仙控制台风格，避免图片化原型。

## Acceptance Criteria

- 打开本地页面即可创建角色并开始行动。
- 输入“修炼”“探索”“拜师”“打听”“炼丹”“挑战”等自然语言能得到不同回合反馈。
- 每回合推进游戏时间并可能触发世界事件。
- 刷新页面后保留存档。
- 可以导出当前修仙人生为文本。
- `npm test` 通过核心引擎测试。
