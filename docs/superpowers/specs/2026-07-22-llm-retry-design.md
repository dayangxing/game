# LLM 调用失败重试设计

## 目标

统一为所有 Bailian LLM 调用增加失败重试：首次调用失败后最多再尝试 3 次，同时保留现有普通 JSON 调用和 SSE 流式剧情体验。

## 方案

- 重试边界放在 `backend/src/llm/bailianClient.js`，让叙事、storyDirector、修复请求共享同一套策略。
- 最多 4 次总尝试：首次调用 + 失败后 3 次重试。
- 重试间隔为 0.5 秒、1 秒、2 秒；测试通过注入等待函数避免真实等待。
- 网络异常、HTTP 408/409/425/429 和 5xx 可重试；API Key 缺失、鉴权失败和其他 4xx 立即失败。
- 普通 JSON 调用会在完整响应收集、空内容或 JSON 解析失败时重试整次请求。
- 流式调用失败后重新建立完整流；如果此前已经向前端发送过片段，后端发送 reset 事件，前端清空半成品后展示下一次尝试，避免文本重复。
- 重试耗尽后沿用现有 fallback / `LLM_UNAVAILABLE` 行为，不改变权威规则结算。

## 影响范围

- 后端 LLM 客户端增加可测试的重试循环和流式重试回调。
- storyDirector、普通 narration 流和 SSE 路由转发 reset 事件。
- 前端 SSE 客户端在 reset 时清理累积 JSON 和当前预览；store 显示“正在重试”状态。

## 非目标

- 不重试规则结算、数据库/存档操作或前端提交请求。
- 不修改模型 prompt、模型选择或后端数值规则。
