# 第 0 回合连续剧情选项设计

## 目标

正式角色完成引导后，第 0 回合也由 `storyDirector` 使用 LLM 生成场景和 2–4 个选项；玩家选择后继续使用现有后端规则结算和 LLM 叙事流程。

## 行为边界

- 新手引导未完成时，继续使用引导行动，不调用 `storyDirector`。
- 正式角色的第 0 回合和后续回合刷新，都优先调用 `storyDirector.invoke({ game, input: { type: 'continue' } })`。
- 只有 `mode === 'choice'` 且至少有 2 个合法选项时，才生成导演行动卡。
- 导演行动使用后端内部 `source: 'director'`，公共响应只暴露玩家可读行动字段和分类元数据。
- 选择导演行动后，复用现有导演效果提示解析、时间推进、规则结算和 LLM 叙事链路。
- LLM 调用失败、返回 `continue` 或选项不足时，回退到现有事件目录行动，不阻塞玩家。
- 同一回合重复刷新时复用尚未消费的导演行动，不重复调用 LLM。

## 验证

- 回归测试确认第 0 回合调用导演并返回公共选项。
- 回归测试确认第 0 回合导演选项可以提交并推进回合。
- 回归测试确认下一回合继续调用导演，同回合刷新不重复调用。
- 保留现有导演失败时事件目录回退测试和完整后端/前端联调测试。
